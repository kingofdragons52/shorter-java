# 🔗 Native Java URL Shortener (High-Performance Focus)

Легковесный и высокопроизводительный сервис для сокращения ссылок. Основная идея проекта — демонстрация работы с протоколом HTTP и базами данных на низком уровне (без использования тяжелых фреймворков типа Spring MVC).

---

## 🚀 Стек технологий
* **Runtime:** Java 21
* **Server:** Нативный `com.sun.net.httpserver` (минимальный overhead).
* **Database:** SQLite (JDBC драйвер `org.xerial:sqlite-jdbc`).
* **UI:** HTML5, Bootstrap 5.

---

## 🛠 Архитектура и алгоритм (Base62)

В отличие от базовых реализаций, использующих случайную генерацию строк и проверку коллизий в цикле `while`, данный сервис использует **детерминированный математический подход**:

1.  **Persistence:** Длинная ссылка сохраняется в БД SQLite, получая уникальный автоинкрементный `id`.
2.  **Encoding:** Полученный `id` преобразуется методом **Base62** (алфавит: `a-z`, `A-Z`, `0-9`).
3.  **Performance:** Этот подход гарантирует полное **отсутствие коллизий** и стабильное время работы алгоритма **$O(1)$** независимо от объема данных в базе.

---

## 🧠 Ключевые особенности
*   **Zero-Dependency Web:** Обработка запросов реализована через кастомные `HttpHandler`. Это позволило глубже изучить жизненный цикл HTTP-запроса, работу с заголовками (`Location`, `Content-Type`) и статус-кодами (302, 404).
*   **Data Integrity:** Использование `PreparedStatements` обеспечивает защиту от SQL-инъекций и корректное экранирование спецсимволов в URL.
*   **Duplicate Check:** Перед созданием новой ссылки сервис проверяет наличие URL в базе, предотвращая избыточное хранение данных.

## 📦 Инструкция по запуску

1. **Сборка проекта:**
   mvn clean package

2. **Запуск:**
    java -cp target/shorter-java-1.0-SNAPSHOT.jar ru.codekitchen.spring.App

    Сервер будет доступен локально на порту 8080: http://127.0.0.1:8080/
